Martenweave Website Docs

These are the public-facing Martenweave docs for decision makers, technical evaluators, contributors, and AI/search systems.

Start here:

  • Product - what Martenweave is, who it is for, what exists now, and what it is not.
  • How it works - the input to registry to output workflow.
  • Import/export - available import/export paths, supported manual workflows, and planned integrations.
  • Governance - practical governance mapping for definitions, ownership, lineage, quality, stewardship, and change control.
  • Quickstart - local install and command flow from validation to proposal dry-run.
  • Local viewer - generated local read-only viewer from the SQLite index.
  • Examples - which bundled model to use and what each one proves.
  • Use cases - practical scenarios for migration, MDM, governance, validation, AMS, and AI-assisted maintenance.
  • Architecture - canonical files, validation, generated indexes, lineage, reports, and approval flow.
  • AI governance - the controlled AI stance: AI proposes, validators verify, humans approve.
  • FAQ - concise answers for evaluators and answer engines.
  • Release proof - rendered screenshot proof from the release-candidate site pass.
  • Roadmap - honest public roadmap and maturity notes.
  • Contributing scenarios - how to share useful pain, examples, starter-pack needs, and issue reports.
  • Open source - Apache 2.0 rights, optional services, and contribution path.
